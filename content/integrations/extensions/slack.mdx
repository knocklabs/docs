The Knock Slack extension lets your team ask questions about your Knock resources directly in Slack. Users can DM the agent or `@Knock` in any channel to kick off an agent session.

## Setting up the Knock Slack extension

1. Go to **[Settings > Extensions](https://dashboard.knock.app/~/settings/integrations/extensions)** in the Knock dashboard.
2. Click "Connect" next to the Slack extension or go to the [installation page](https://control.knock.app/slack/install) here.
3. You'll be prompted to install the Knock app for Slack in your workspace.
4. After installing, you'll be redirect to Knock to finalize the installation.
5. Start using the Slack extension by prompting the Knock agent in a direct message or `@Knock` in any channel.

## Authenticating users

In order to use the Knock Slack extension, users will need to link their Slack user to their Knock account.

Users within your Slack workspace can authenticate by prompting the Knock agent in a direct message or `@Knock` in any channel. They will receive a DM with a button to authenticate. Clicking the button will take them to the Knock dashboard where they can authenticate their account.

Once linked, users can then disconnect their linked Slack account from their user settings under "Settings > Profile > Overview". Once disconnected, they will no longer be able to use the Knock Slack extension.

## Using the Knock Slack extension

### Kicking off an agent

Users can kick off an agent session by prompting the Knock agent in a direct message or `@Knock` in any channel. The agent will respond with a message indicating that it has started and will begin processing the user's request.

The Knock app will only be available in the public channels that it has been invited to. You can invite the Knock app to a channel by at-mentioning it in the channel, or using the "Add to channel" button in Slack.

Here are some example prompts you can use to kick off an agent session:

- `@Knock create a new onboarding workflow that sends a welcome compaign to new users`
- `@Knock create a new audience of paid subscribers`
- `@Knock find all of my workflows that are sending emails`

### Modified resources

If the agent session modifies one or more resources, you will see a link to the resource in the conversation history. Clicking the link will open the resource in the Knock dashboard.

### Specifying an environment

By default, the Knock Slack extension will operate in the `development` environment. You can specify a different environment by referencing it within your prompt. The Knock agent will attempt to automatically determine the environment based on the context of the prompt.

As an example, you can specify the environment in a prompt like this:

> @Knock in the production environment, update the subject line of the welcome email to 'Welcome to our platform!'

### Viewing an agent session

From a kicked off agent session, you can view the conversation history by clicking the "Open agent" button. This will open a new tab with the conversation history in the Knock dashboard. Any user within your Slack workspace can view the conversation history of an agent session by clicking the "Open agent" button.
