The `useSlackChannels` hook is used to retrieve available Slack channels for a connected workspace. This hook must be used within a `KnockSlackProvider`.

## Parameters

Accepts an object with the following properties:

<Attributes>
  <Attribute
    name="queryOptions"
    type="SlackChannelQueryOptions"
    description="Optional query options for filtering and pagination. Includes limitPerPage, types, and cursor."
  />
</Attributes>

## Returns

Returns an object with the following properties:

<Attributes>
  <Attribute
    name="data"
    type="SlackChannel[]"
    description="Array of available Slack channels."
  />
  <Attribute
    name="isLoading"
    type="boolean"
    description="Whether channels are being loaded or revalidating."
  />
  <Attribute
    name="refetch"
    type="() => void"
    description="Function to manually refetch the channels."
  />
</Attributes>

## Example

### Basic usage

The following example demonstrates how to use the `useSlackChannels` hook to display available Slack channels.

```tsx
import { KnockSlackProvider, useSlackChannels } from "@knocklabs/react";

const SlackChannelList = () => {
  const { data: channels, isLoading, refetch } = useSlackChannels({});

  if (isLoading) return <div>Loading channels...</div>;

  return (
    <div>
      <h2>Available Channels</h2>
      <button onClick={refetch}>Refresh</button>
      <ul>
        {channels.map((channel) => (
          <li key={channel.id}>{channel.name}</li>
        ))}
      </ul>
    </div>
  );
};

const App = () => (
  <KnockSlackProvider
    knockSlackChannelId="slack-channel-id"
    tenantId="tenant-id"
  >
    <SlackChannelList />
  </KnockSlackProvider>
);
```

### With query options

You can customize the query with pagination and filtering options:

```tsx
const { data: channels } = useSlackChannels({
  queryOptions: {
    limitPerPage: 50,
    types: ["public_channel", "private_channel"],
  },
});
```
